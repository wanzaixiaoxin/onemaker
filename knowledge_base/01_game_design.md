# 01 游戏设计（Game Design）

## 一、概念设计（Concept Design）

### 职责定义
确定游戏的核心概念、类型、目标受众、差异化卖点。这是一切设计工作的起点。

### 关键产出物
- **概念文档（Concept Document）**：1-2页，概述游戏核心想法
- **电梯演讲（Elevator Pitch）**：一句话描述你的游戏
- **参考图板（Moodboard）**：视觉/氛围参考

### 工作方法
1. 确定游戏类型（动作/RPG/策略/模拟/叙事/解谜/...）
2. 定义核心体验关键词（3-5个词，如"紧张、探索、孤独"）
3. 找到差异化卖点（USP）：为什么玩家要玩你的游戏？
4. 确定目标平台和目标受众
5. 评估可行性（独立开发者的资源限制）

### AI协作方式
- **头脑风暴**：让AI根据类型生成创意方向
- **概念文档撰写**：给AI关键词，让它生成概念文档初稿
- **竞品分析**：让AI帮你梳理同类型游戏的优缺点
- **市场定位**：让AI分析目标受众画像

### AI Prompt模板
```
我正在设计一款[类型]游戏，核心体验是[关键词]，
目标平台是[平台]，目标受众是[受众]。
请帮我：
1. 生成5个有差异化的核心概念
2. 为每个概念写一个电梯演讲
3. 分析每个概念的可行性和风险
```

## 二、核心玩法设计（Core Gameplay Design）

### 职责定义
设计游戏的核心循环（Core Loop）、操作方式、进度系统、难度曲线。核心玩法是玩家持续游玩的动力。

### 关键产出物
- **游戏设计文档（GDD - Game Design Document）**
- **核心循环图**：玩家行为的循环流程
- **操作映射表**：输入 → 行为的完整映射
- **数值表**：伤害、血量、经验值等数值设计

### 核心循环模型
```
探索 → 发现 → 奖励 → 成长 → 更强的探索能力 → 探索（更深层）
```
每个游戏都需要一个这样的循环，让玩家不断投入。

### 核心玩法三要素
1. **动作（Action）**：玩家做什么？（跳跃/射击/建造/对话/...）
2. **反馈（Feedback）**：玩家做之后发生什么？（视觉/音效/数值变化）
3. **动机（Motivation）**：为什么玩家要继续？（好奇心/成就感/社交/...）

### 数值设计要点
- 建立数值框架：基础值、成长系数、上限
- 难度曲线：线性/指数/S型
- 心流模型：难度与玩家技能的平衡（Mihaly Csikszentmihalyi）
- 边际递减：防止玩家过度集中某一策略

### AI协作方式
- **核心循环设计**：描述体验目标，让AI生成核心循环方案
- **GDD撰写**：提供设计要点，让AI生成结构化GDD
- **数值平衡**：提供数值框架，让AI计算平衡性
- **难度曲线**：描述目标难度感觉，让AI生成数值曲线

### AI Prompt模板
```
我的游戏核心循环是[描述循环]，
玩家通过[动作]获得[奖励]，然后[成长]。
请帮我：
1. 设计完整的数值框架（等级/伤害/血量/经验值）
2. 画出难度曲线的建议（前10小时的游戏体验）
3. 找出核心循环可能的断裂点
```

## 三、关卡设计（Level Design）

### 职责定义
设计关卡的空间布局、节奏、流程、隐藏要素。关卡是核心玩法的容器。

### 关键产出物
- **关卡设计文档**：每个关卡的设计意图和流程
- **白盒原型**：用简单几何体搭建的关卡骨架
- **流程图**：玩家在关卡中的路径和分支
- **节奏图**：关卡内紧张度随时间的变化曲线

### 关卡设计原则
1. **引导（Guidance）**：用灯光、颜色、构图引导玩家前进
2. **节奏（Pacing）**：紧张与放松的交替（Valve的30秒法则）
3. **教学（Tutorial）**：安全区展示机制 → 简单挑战 → 复杂组合
4. **奖励（Reward）**：探索奖励、节奏奖励、成就感奖励
5. **重玩性（Replayability）**：多路径、随机要素、评分系统

### 关卡设计流程
1. 确定关卡目的（教学/挑战/叙事/过渡）
2. 画出流程图（主路径+支路径+秘密区域）
3. 白盒搭建（不使用美术资源，纯几何体验）
4. 反复测试和迭代
5. 替换为正式美术资产

### AI协作方式
- **关卡概念生成**：描述关卡目的，让AI生成布局概念
- **节奏设计**：让AI根据关卡时长生成节奏曲线
- **谜题设计**：让AI基于已有机制生成谜题变体
- **关卡描述文档**：让AI将白盒截图转化为设计文档

## 四、系统设计（System Design）

### 职责定义
设计游戏的元系统：经济系统、成长系统、装备系统、任务系统等。系统设计让游戏有深度和广度。

### 关键产出物
- **系统规格文档**：每个系统的详细规则
- **系统交互图**：系统之间的关系和数据流
- **配置表**：可调节的系统参数

### 常见游戏系统
| 系统 | 核心要素 | 设计要点 |
|------|---------|---------|
| 经济系统 | 货币、商店、产出、消耗 | 通胀控制、产出消耗平衡 |
| 成长系统 | 等级、技能、属性 | 成长节奏、选择深度 |
| 装备系统 | 武器、防具、饰品 | 稀有度、套装效果、差异化 |
| 任务系统 | 主线、支线、日常 | 任务链设计、奖励分配 |
| 成就系统 | 成就、收集、挑战 | 引导探索、增加粘性 |
| 天赋/技能树 | 节点、分支、解锁条件 | 构建多样性、避免最优解 |

### 系统设计原则
1. **涌现（Emergence）**：简单规则组合出复杂行为
2. **风险/回报**：每个选择都有代价
3. **可见深度**：新手易懂，老手有深度
4. **避免陷阱选项**：每个选择都应有合理场景

### AI协作方式
- **系统规则生成**：描述系统目标，让AI生成规则草案
- **数值推演**：让AI模拟经济系统的运行
- **系统交互分析**：让AI找出系统间的冲突和协同
- **配置表生成**：让AI根据规则生成初始配置数据

## 五、叙事设计（Narrative Design）

### 职责定义
设计游戏的世界观、剧情、角色、对话。叙事为游戏提供情感深度。

### 关键产出物
- **世界观设定文档**：历史、地理、文化、种族
- **角色设定文档**：角色背景、性格、动机、关系图
- **剧情大纲**：主线剧情的章节/幕结构
- **对话脚本**：NPC对话、过场台词、环境叙事文本
- **叙事图表**：分支剧情的流程图

### 叙事设计方法
1. **三幕结构**：建置 → 对抗 → 解决
2. **英雄之旅**（Joseph Campbell）：12步骤的经典叙事
3. **环境叙事**：通过场景物品、文字碎片讲述故事
4. **分支叙事**：基于玩家选择的多结局设计
5. **碎片化叙事**：散落的信息片段，玩家自行拼凑

### 独立开发者叙事策略
- 量力而行：独立游戏叙事宜精不宜多
- 用玩法讲故事：Show, don't tell
- 环境叙事成本低效果好
- AI可以大幅降低文案写作成本

### AI协作方式
- **世界观构建**：给AI几个关键词，生成完整世界观
- **角色设计**：描述角色原型，让AI生成详细设定
- **对话撰写**：提供角色性格和场景，让AI生成对话选项
- **剧情分支**：描述关键决策点，让AI设计后果链
- **文本润色**：让AI优化已有文案的表达

### AI Prompt模板
```
游戏世界观：[描述]
角色：[名字]，性格[描述]，动机[描述]
当前场景：[描述]
请为这个场景生成3个对话选项，每个选项体现角色不同的态度，
并描述每个选择可能的后果。
```

## 设计阶段检查清单

完成设计阶段前，确认以下事项：

- [ ] 电梯演讲是否清晰？
- [ ] 核心循环是否自洽？
- [ ] 是否有可运行的最小原型？
- [ ] 数值框架是否搭建完成？
- [ ] 第一个关卡是否完成白盒？
- [ ] 关键系统的规则是否文档化？
- [ ] 剧情大纲是否完成？
- [ ] GDD是否更新到最新状态？
