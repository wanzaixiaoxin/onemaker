# 02 程序开发（Programming）

## 一、引擎/框架选型（Engine & Framework Selection）

### 职责定义
选择最适合项目需求的游戏引擎、编程语言和工具链。这是技术决策的基石。

### 主流引擎对比

| 引擎 | 语言 | 适合类型 | 优势 | 劣势 | AI辅助开发成熟度 |
|------|------|---------|------|------|-----------------|
| Unity | C# | 全类型 | 生态丰富、2D/3D均强 | 抽水制度变化、性能上限 | ★★★★★ |
| Unreal Engine | C++/蓝图 | 3A、写实3D | 渲染顶级、蓝图可视化 | 学习曲线陡、包体大 | ★★★★ |
| Godot | GDScript/C# | 2D/轻量3D | 开源免费、轻量 | 生态较小、3D待成熟 | ★★★ |
| Cocos Creator | TypeScript | 2D/小游戏 | 中国生态、小游戏适配 | 3D能力弱 | ★★★ |
| 自研引擎 | C++/Rust | 特殊需求 | 完全定制 | 开发成本极高 | ★★ |

### 选型决策树
```
你的游戏是？
├── 2D为主
│   ├── 需要发布到微信小游戏？ → Cocos Creator
│   ├── 追求轻量和开源？ → Godot
│   └── 需要丰富生态？ → Unity
├── 3D写实风格
│   ├── 追求顶级画质？ → Unreal Engine
│   └── 平衡画质和效率？ → Unity
└── 特殊需求（大量定制渲染/物理）
    └── 考虑自研引擎（慎选）
```

### 独立开发者推荐
- **首选 Unity**：社区最大，AI训练数据最多，遇到问题最容易找到答案
- **2D游戏可选 Godot**：轻量、开源、无商业限制
- **避免频繁换引擎**：选定后坚持，切换成本极高

### AI协作方式
- **技术选型分析**：描述项目需求，让AI对比引擎优劣
- **架构建议**：让AI根据引擎特性推荐架构模式

## 二、核心架构设计（Core Architecture）

### 职责定义
搭建游戏代码的整体架构，定义模块划分、数据流向、设计模式。好的架构让开发事半功倍。

### 关键产出物
- **架构图**：模块关系图
- **编码规范文档**：命名、目录结构、代码风格
- **核心框架代码**：基础类和工具

### 常见架构模式

#### 1. MVC（Model-View-Controller）
- Model：游戏数据和逻辑
- View：显示和渲染
- Controller：输入处理和协调
- 适合：UI密集型游戏

#### 2. ECS（Entity-Component-System）
- Entity：实体（纯ID）
- Component：组件（纯数据）
- System：系统（纯逻辑）
- 适合：大量同类对象的游戏（RTS、Roguelike）

#### 3. 状态机（FSM）
- 定义对象在不同状态间的切换
- 适合：角色AI、UI状态、游戏流程

#### 4. 事件总线（Event Bus）
- 模块间松耦合通信
- 适合：全局事件通知（成就、任务触发等）

#### 5. 对象池（Object Pool）
- 复用对象，减少GC压力
- 适合：子弹、特效、敌人等高频创建销毁的对象

### 推荐项目结构（Unity示例）
```
Assets/
├── Scripts/
│   ├── Core/           # 核心框架（单例管理器、事件系统、存档系统）
│   ├── Gameplay/       # 游戏逻辑（角色、敌人、道具、武器）
│   ├── AI/             # AI行为树、状态机
│   ├── UI/             # UI面板、HUD
│   ├── Data/           # 数据结构、ScriptableObject
│   ├── Utils/          # 工具类、扩展方法
│   └── Config/         # 配置、常量
├── Prefabs/            # 预制体
├── Art/                # 美术资源
├── Audio/              # 音频资源
└── Scenes/             # 场景文件
```

### AI协作方式
- **架构设计**：描述游戏需求，让AI推荐架构方案
- **设计模式选择**：描述具体问题，让AI推荐合适的设计模式
- **代码框架生成**：让AI生成基础框架代码
- **代码审查**：让AI审查代码质量和架构合理性

## 三、玩法实现（Gameplay Implementation）

### 职责定义
实现游戏的核心玩法逻辑，包括角色控制、物理系统、碰撞检测、战斗系统等。

### 关键模块

#### 角色控制器（Character Controller）
- 输入处理：键盘/手柄/触屏
- 移动逻辑：走/跑/跳/冲刺/攀爬
- 动画驱动：基于状态的动画切换
- 物理交互：与环境的物理响应

#### 战斗系统
- 伤害计算：攻击力、防御力、暴击、属性克制
- 碰撞检测：近战判定（射线/球体/盒体）、远程弹道
- 状态效果：中毒、冰冻、眩晕、击退
- AI敌人：行为树/状态机驱动

#### 物理系统
- 刚体运动：重力、摩擦、弹性
- 触发器：进入/停留/离开事件
- 射线检测：瞄准、视线、地面检测

#### 摄像机系统
- 跟随摄像机：平滑跟随、滞后
- 摄像机震动：受伤/爆炸反馈
- 多摄像机切换：过场/游戏/小地图

### 性能优化要点
- **避免每帧new对象**：使用对象池
- **减少Find调用**：缓存引用
- **合批渲染**：减少Draw Call
- **LOD**：远处使用低精度模型
- **遮挡剔除**：不渲染不可见物体
- **异步加载**：场景/资源的异步加载

### AI协作方式
- **算法实现**：描述需求，让AI生成具体算法代码
- **Bug修复**：提供错误信息和上下文，让AI定位修复
- **性能优化**：让AI分析代码瓶颈并优化
- **单元测试**：让AI为核心逻辑生成测试代码

## 四、UI/UX开发（UI/UX Development）

### 职责定义
实现游戏的用户界面系统，包括菜单、HUD、弹窗、设置等。

### 关键产出物
- **UI系统架构**：面板管理器、动画系统
- **UI预制体**：各界面组件
- **UX交互规范**：交互反馈标准

### UI系统模块
| 模块 | 功能 | 设计要点 |
|------|------|---------|
| 主菜单 | 开始/继续/设置/退出 | 简洁、快速进入游戏 |
| HUD | 血条/弹药/小地图/提示 | 信息层级清晰、不遮挡视野 |
| 背包/装备 | 物品管理、装备穿戴 | 拖拽操作、信息密度 |
| 对话系统 | NPC对话、选择分支 | 打字机效果、选项高亮 |
| 设置面板 | 音量/画质/按键绑定 | 即时生效、可重置 |
| 成就面板 | 成就列表、进度 | 解锁动画、分类展示 |

### UI架构设计
```csharp
// UI面板管理器（简化示例）
public class UIManager : Singleton<UIManager> {
    // 打开面板
    public T Open<T>() where T : UIPanel;
    // 关闭面板
    public void Close<T>() where T : UIPanel;
    // 关闭所有
    public void CloseAll();
    // 显示提示
    public void ShowToast(string msg);
}
```

### UX设计原则
1. **响应性**：所有交互在100ms内给出视觉反馈
2. **一致性**：同类操作使用相同的交互方式
3. **容错性**：重要操作需要确认，支持撤销
4. **可达性**：支持键鼠/手柄/触屏多种输入
5. **可读性**：字体大小、颜色对比度符合标准

### AI协作方式
- **UI布局代码**：描述界面需求，让AI生成布局代码
- **交互逻辑**：让AI实现面板切换、数据绑定逻辑
- **动画效果**：让AI编写UI动画脚本
- **适配方案**：让AI处理多分辨率适配

## 五、工具链/管线（Toolchain & Pipeline）

### 职责定义
构建开发自动化工具、资源管线、编辑器扩展，提升开发效率。

### 关键工具
| 工具 | 用途 | 优先级 |
|------|------|-------|
| 编辑器扩展 | 自定义Inspector、工具窗口 | 高 |
| 自动构建 | 一键打包多平台 | 高 |
| 数据导表 | Excel/JSON → 游戏配置 | 高 |
| 资源检查 | 检查资源规范（大小、格式） | 中 |
| CI/CD | 自动化测试和构建 | 中 |
| 热更新 | 运行时更新游戏内容 | 低（按需） |

### 自动化构建流程
```
代码提交 → 自动编译 → 自动测试 → 自动打包 → 产出构建包
```

### AI协作方式
- **编辑器脚本**：描述需要的编辑器功能，让AI生成扩展代码
- **构建脚本**：让AI编写自动构建pipeline
- **数据转换工具**：让AI编写配置表导出工具
- **代码生成器**：让AI根据配置自动生成数据类代码
