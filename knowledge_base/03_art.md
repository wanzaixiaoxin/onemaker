# 03 美术资源（Art）

## 一、视觉风格定义（Visual Style Definition）

### 职责定义
确定游戏的整体美术方向、色彩方案、参考标准。风格定义是所有美术工作的指南针。

### 关键产出物
- **美术风格指南（Art Bible）**：风格定义、色彩规范、比例标准
- **参考图板（Reference Board）**：Pinterest风格的情绪板
- **关键概念图（Key Art）**：1-2张代表游戏视觉的标志性图像

### 常见游戏美术风格
| 风格 | 代表作 | 制作难度 | AI生成适配度 |
|------|-------|---------|-------------|
| 像素风 | Celeste、星露谷 | ★★ | ★★★★ |
| 低多边形 | Henry Stickmin | ★★ | ★★★★★ |
| 手绘水彩 | 饥荒、GRIS | ★★★ | ★★★ |
| 卡通渲染 | 塞尔达BotW、原神 | ★★★ | ★★★★ |
| 写实风 | 荒野大镖客2 | ★★★★★ | ★★★ |
| 体素风 | 我的世界、Teardown | ★★ | ★★★★ |
| 赛博朋克 | 赛博朋克2077 | ★★★★ | ★★★★★ |
| 极简风 | Journey、纪念碑谷 | ★★ | ★★★ |

### 独立开发者风格选择原则
1. **风格统一比质量重要**：低精度但统一 >> 高精度但混乱
2. **选择AI擅长的风格**：像素、低模、卡通渲染等
3. **控制资产数量**：用模块化设计减少资产总量
4. **复用和变体**：一套基础资产通过调色/组合产生变化

### AI协作方式
- **风格探索**：让AI生成多种风格的concept art
- **色彩方案**：描述情绪关键词，让AI生成配色
- **参考图收集**：让AI推荐同风格的参考游戏和艺术家
- **Art Bible撰写**：让AI帮你编写风格指南文档

### AI Prompt模板
```
我正在制作一款[类型]游戏，世界观是[描述]，
希望美术风格是[风格关键词]，
请帮我：
1. 生成5个视觉方向的概念描述
2. 为每个方向推荐配色方案（主色/辅色/强调色）
3. 列出需要收集的参考图类别
```

## 二、2D/3D资产制作（Asset Production）

### 职责定义
制作游戏所需的所有视觉资产：角色、场景、道具、特效。

### 2D资产管线

#### 角色制作流程
```
概念设定 → 线稿 → 上色 → 分层/切帧 → 导入引擎
```

#### 场景制作流程
```
场景概念 → 素描稿 → 分层绘制 → 平铺/视差 → 导入引擎
```

#### 像素画特别技巧
- 画布尺寸：角色通常32x32 ~ 64x64
- 调色板限制：16色或32色，保持风格统一
- 动画帧数：走路4-6帧，攻击4-8帧
- 工具推荐：Aseprite、PyxelEdit

### 3D资产管线

#### 角色制作流程
```
概念设定 → 原画 → 建模 → UV展开 → 材质贴图 → 绑骨 → 动画 → 导入引擎
```

#### 场景制作流程
```
场景概念 → 白盒 → 建模 → 材质 → 布光 → 优化 → 导入引擎
```

#### 3D资产规格参考
| 资产类型 | 面数范围 | 贴图尺寸 | 骨骼数 |
|---------|---------|---------|-------|
| 主角 | 5K-15K | 2048x2048 | 50-100 |
| NPC | 3K-8K | 1024x1024 | 30-60 |
| 场景道具 | 100-2K | 256-512 | 无 |
| 场景建筑 | 2K-10K | 1024-2048 | 无 |
| 特效 | 面片为主 | 256-512 | 无 |

| Mixamo | 自动绑骨和动画 | ★★★★★ |
| Blockade Labs | 天空盒生成 | ★★★★ |

### 模块化设计策略
```
一套基础资产 → 调色变体 × 组合变体 × 尺寸变体 = 大量视觉变化

示例：
- 1个墙壁模型 × 3种材质 × 4种破损程度 = 12种墙壁
- 1个树模型 × 3种季节色 × 3种大小 = 9棵不同的树
```

### AI协作方式
- **概念图生成**：描述角色/场景，AI生成多方案concept
- **贴图生成**：AI生成材质贴图（法线、粗糙度、AO等）
- **像素画辅助**：AI生成像素画基础，手动调整
- **3D模型参考**：AI生成3D模型草稿，手动精修
- **资产变体**：AI基于基础资产生成调色/风格变体

## 三、UI美术（UI Art）

### 职责定义
设计游戏的用户界面视觉，包括布局、图标、字体、动画效果。

### 关键产出物
- **UI风格指南**：按钮/面板/字体的视觉规范
- **UI切图包**：.9图/精灵图集
- **图标集**：道具/技能/成就图标
- **UI动画规格**：转场/弹出/反馈动画说明

### UI设计流程
1. 线框图（Wireframe）→ 确定布局和信息层级
2. 视觉稿（Mockup）→ 套用美术风格
3. 切图（Export）→ 导出引擎可用的格式
4. 交互动画（Animation）→ 设计转场和反馈动画
5. 实装测试 → 在引擎中验证效果

### UI设计原则
- **层级分明**：重要信息 > 次要信息 > 装饰元素
- **一致性**：同功能元素风格统一
- **可读性**：文字与背景对比度足够
- **触控友好**：按钮最小44x44pt
- **适配性**：支持16:9/16:10/21:9等多种比例

### AI协作方式
- **UI概念设计**：描述UI功能，让AI生成布局建议
- **图标生成**：用AI生成道具/技能图标
- **配色方案**：让AI生成符合游戏风格的UI配色
- **字体推荐**：让AI推荐适合游戏风格的字体

## 四、技术美术（Technical Art / TA）

### 职责定义
编写Shader、优化渲染管线、制作美术工具、解决美术与程序之间的技术问题。

### 关键产出物
- **Shader库**：常用材质Shader（水、玻璃、描边、溶解等）
- **后处理配置**：色彩校正、泛光、景深、抗锯齿
- **美术工具脚本**：批量处理、规范检查
- **性能优化方案**：LOD、合批、纹理压缩

### 常用Shader效果
| 效果 | 使用场景 | 实现难度 |
|------|---------|---------|
| 描边（Outline） | 角色选中/高亮 | ★★ |
| 溶解（Dissolve） | 消失/传送特效 | ★★ |
| 水面（Water） | 场景水面 | ★★★ |
| 卡通渲染（Toon） | NPR风格 | ★★★ |
| 粒子特效 | 战斗/环境 | ★★ |
| 屏幕后效 | 全局氛围 | ★★★ |

### 渲染管线选择（Unity）
- **Built-in**：传统管线，兼容性好
- **URP**：通用渲染管线，推荐大多数项目使用
- **HDRP**：高清渲染管线，适合写实3D

### 性能优化检查点
- [ ] Draw Call数量 < 100（移动端）/ < 500（PC）
- [ ] 面数总量合理（移动端 < 10万，PC < 100万）
- [ ] 贴图总内存 < 预算
- [ ] SetPass Call最小化
- [ ] 使用纹理图集（Atlas）
- [ ] 合理使用LOD

### AI协作方式
- **Shader编写**：描述效果需求，让AI生成Shader代码
- **渲染问题排查**：提供截图，让AI分析渲染异常
- **性能分析**：提供Profiler数据，让AI定位瓶颈
- **工具脚本**：让AI编写批量处理工具
