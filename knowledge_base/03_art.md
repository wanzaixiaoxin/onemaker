# 03 美术资源（Art）

## 一、视觉风格定义（Visual Style Definition）

### 职责定义
确定游戏的整体美术方向、色彩方案、参考标准。风格定义是所有美术工作的指南针。

### 关键产出物
- **美术风格指南（Art Bible）**：风格定义、色彩规范、比例标准
- **参考图板（Reference Board）**：Pinterest风格的情绪板

### 详细操作步骤

#### Step 1: 确定视觉风格类型
| 风格 | 特征 | 适合游戏类型 | 制作难度 | AI生成适合度 |
|------|------|-------------|---------|-------------|
| 像素风 | 低分辨率、复古感 | Roguelike、平台跳跃 | 低 | ★★★★★ |
| 卡通渲染 | 描边+色块 | 全类型 | 中 | ★★★★ |
| 手绘风 | 水彩/油画质感 | 冒险、叙事 | 中高 | ★★★★ |
| Low-Poly | 低面数几何 | 独立游戏、策略 | 低 | ★★★★★ |
| 写实风 | PBR材质、真实光影 | FPS、RPG | 高 | ★★★ |
| 赛博朋克 | 霓虹+暗色调 | 动作、RPG | 中 | ★★★★ |
| 水墨风 | 国画留白、晕染 | 武侠、国风 | 中 | ★★★★ |

#### Step 2: 制作色彩方案
```
色彩方案模板：

主色调（Primary）: #______ 占比60% （背景、大面积）
辅助色（Secondary）: #______ 占比30% （角色、UI元素）
强调色（Accent）: #______ 占比10% （重要提示、互动元素）

暗部色: #______ （阴影、深度）
亮部色: #______ （高光、前景）
警示色: #______ （危险、HP低）
安全色: #______ （回复、友好NPC）

色彩情绪关键词：____________
参考作品：____________
```

#### Step 3: 建立视觉规范文档
```
Art Bible 必备内容：

1. 角色比例标准
   - 主角身高 = X像素/单位
   - NPC身高比例范围
   - 头身比规范（如Q版3头身）

2. 线条规范
   - 描边宽度：__px
   - 描边颜色：#______
   - 是否统一描边

3. 阴影规范
   - 光源方向：左上/右上/顶部
   - 阴影色：主色相 + 偏移 __ 色阶
   - 阴影层数：__ 层

4. 材质规范
   - 金属：高光+环境反射
   - 布料：柔和渐变
   - 皮肤：次表面散射感

5. 动画帧率标准
   - 角色动画：__ FPS
   - 特效动画：__ FPS
   - UI动画：__ FPS
```

### AI协作方式
- **风格探索**：用AI图像生成工具快速尝试多种风格方向
- **参考图生成**：用Midjourney/Stable Diffusion生成情绪板素材
- **配色方案**：让AI根据风格关键词推荐配色

## 二、2D美术制作（2D Art Production）

### 职责定义
制作游戏所需的所有2D图形资源：角色立绘/精灵、场景背景、UI元素、图标、特效帧。

### 关键产出物
- **角色精灵表（Sprite Sheet）**：角色动画帧序列
- **场景瓦片集（Tileset）**：可复用的场景元素
- **UI素材包**：按钮、面板、图标
- **特效帧序列**：粒子、爆炸、魔法效果

### 详细操作步骤

#### Step 1: 角色制作流程

##### 像素角色制作规范
```
角色尺寸标准表：

| 类型 | 尺寸(px) | 帧数(走) | 帧数(攻击) |
|------|---------|---------|-----------|
| 主角 | 32×32 | 4-6帧 | 4-8帧 |
| 主角(大) | 64×64 | 6-8帧 | 6-12帧 |
| NPC | 24×32 | 4帧 | - |
| Boss | 128×128+ | 8-12帧 | 10-20帧 |
| 小怪 | 16×16 | 3-4帧 | 3-6帧 |

动画列表（最小集）：
- [ ] Idle（待机） 4帧循环
- [ ] Walk（行走） 4-6帧循环
- [ ] Run（奔跑） 4-6帧循环
- [ ] Jump（跳跃） 3帧：起跳/滞空/落地
- [ ] Attack（攻击） 4-6帧单次
- [ ] Hurt（受击） 2帧单次
- [ ] Death（死亡） 3-5帧单次
```

##### AI辅助角色制作流程
```
1. 概念阶段：用AI生成角色概念图
   Prompt模板：
   "game character sprite, [类型] [风格], [描述],
    white background, front view, pixel art"

2. 线稿阶段：基于概念图细化线稿
   - 使用Aseprite/Photoshop描线
   - 保持线条简洁、可辨识

3. 上色阶段：
   - 填充基础色块
   - 按Art Bible加阴影层
   - 加高光点

4. 动画阶段：
   - 在Aseprite中逐帧调整
   - 使用洋葱皮(Onion Skin)辅助
   - 导出为Sprite Sheet
```

#### Step 2: 场景/瓦片制作

##### 瓦片集制作规范
```
瓦片尺寸标准：
- 标准：16×16 / 32×32 / 48×48
- 大场景：64×64

瓦片集结构（以32×32为例）：
┌──────┬──────┬──────┬──────┐
│ 地面 │ 地面 │ 地面 │ 地面 │
│ 平面 │ 上边 │ 左边 │ 外角 │
├──────┼──────┼──────┼──────┤
│ 墙壁 │ 墙壁 │ 墙壁 │ 内角 │
│ 平面 │ 上边 │ 左边 │      │
├──────┼──────┼──────┼──────┤
│ 装饰 │ 装饰 │ 装饰 │ 装饰 │
│ 草地 │ 花朵 │ 石头 │ 树桩 │
└──────┴──────┴──────┴──────┘

必须包含的边缘类型：
- [ ] 平面（Flat）
- [ ] 单边（Edge Top/Bottom/Left/Right）
- [ ] 外角（Outer Corner）
- [ ] 内角（Inner Corner）
- [ ] T型交叉（T-Junction）
```

##### 场景分层
```
推荐分层结构（从后到前）：

Layer 0: 天空/远景（不滚动或慢速视差）
Layer 1: 中景建筑/山体（中速视差）
Layer 2: 地面/平台（1:1滚动）
Layer 3: 前景装饰（快速视差）
Layer 4: 特效/天气（最前层）

视差滚动公式：
layer_offset = camera_x * parallax_factor
parallax_factor:
  远景 = 0.1 ~ 0.3
  中景 = 0.5 ~ 0.7
  地面 = 1.0
  前景 = 1.1 ~ 1.3
```

#### Step 3: UI素材制作
```
UI素材规范表：

| 类型 | 尺寸 | 格式 | 要求 |
|------|------|------|------|
| 按钮 | 9-Slice | PNG | 四角圆角，可拉伸 |
| 面板背景 | 9-Slice | PNG | 半透明+边框 |
| 图标 | 64×64/128×128 | PNG | 统一描边风格 |
| 字体 | SDF Bitmap | PNG+XML | ASCII+中文常用字 |
| HP/MP条 | 9-Slice | PNG | 前景+背景+边框 |

9-Slice切割规范：
┌───┬───────────┬───┐
│ 1 │     2     │ 3 │  ← 角（不拉伸）
├───┼───────────┼───┤
│ 4 │     5     │ 6 │  ← 边（单轴拉伸）
├───┼───────────┼───┤
│ 7 │     8     │ 9 │  ← 角（不拉伸）
└───┴───────────┴───┘
中心(5)：双轴拉伸
```

#### Step 4: 特效帧制作
```
特效类型与参数：

| 特效 | 帧数 | 尺寸 | 说明 |
|------|------|------|------|
| 爆炸 | 8-12帧 | 64×64~128×128 | 从中心扩散 |
| 斩击 | 4-6帧 | 64×64 | 弧形轨迹 |
| 魔法弹 | 4帧循环 | 32×32 | 发光+拖尾 |
| 治疗 | 6-8帧 | 64×64 | 向上粒子 |
| 烟雾 | 6-8帧 | 32×32 | 扩散+消散 |

特效制作要点：
- 使用加法混合(Additive Blending)增强发光感
- 首帧和末帧要自然过渡
- 提供中心锚点坐标
```

### 2D工具推荐
| 工具 | 用途 | 价格 | AI支持 |
|------|------|------|-------|
| Aseprite | 像素画/动画 | 付费 | ★★ |

## 三、3D美术制作（3D Art Production）

### 职责定义
制作3D模型、材质、骨骼动画、场景搭建。

### 关键产出物
- **3D模型**：角色、道具、场景物件
- **材质贴图**：PBR材质球
- **骨骼动画**：角色动作
- **场景布局**：关卡场景搭建

### 详细操作步骤

#### Step 1: 3D角色制作管线
```
完整管线：

1. 概念设定 → AI生成概念图
2. 白模（Blockout）→ Blender基础几何体
3. 高模雕刻 → ZBrush/Blender Sculpt
4. 拓扑（Retopo）→ Quad Draw / Blender
5. UV展开 → Blender UV Editor
6. 烘焙（Bake）→ 高模细节 → 低模贴图
7. 材质绘制 → Substance Painter
8. 骨骼绑定 → Blender Rigging
9. 动画制作 → Blender Animation
10. 导出 → FBX → 引擎导入

面数预算：
| 类型 | 三角面数 | 贴图尺寸 |
|------|---------|---------|
| 主角 | 5K-15K | 2048×2048 |
| NPC | 2K-8K | 1024×1024 |
| 小怪 | 500-3K | 512×512 |
| Boss | 15K-50K | 2048×2048 |
| 场景道具 | 100-2K | 256-512 |
| 场景建筑 | 5K-30K | 1024-2048 |
```

#### Step 2: PBR材质制作
```
PBR贴图通道：

Albedo/Base Color（基础色）
  - 无光照信息，纯色
  - 无高光/阴影

Normal Map（法线贴图）
  - 增加表面细节凹凸
  - 从高模烘焙生成

Metallic/Roughness（金属度/粗糙度）
  - Metallic: 0=非金属 1=金属
  - Roughness: 0=光滑反射 1=粗糙漫反射

AO（环境光遮蔽）
  - 模型缝隙处的暗影
  - 增加立体感

Emission（自发光）
  - 发光材质（霓虹、火焰）
  - 需配合Bloom后处理

Substance Painter工作流：
1. 烘焙贴图（高模→低模）
2. 填充基础材质
3. 添加智能蒙版（边缘磨损、污垢）
4. 手绘细节
5. 导出为引擎格式
```

#### Step 3: 骨骼动画制作
```
骨骼规范：

标准人形骨骼（最少集）：
- Root（根骨骼）
- Hips（臀部）
- Spine×3（脊椎）
- Neck（颈部）
- Head（头部）
- L/R Shoulder → Arm → Forearm → Hand
- L/R UpperLeg → LowerLeg → Foot

动画帧率：30FPS
导出格式：FBX（勾选Bake Animation）

必须制作的动画列表：
| 动画 | 帧数 | 循环 | 优先级 |
|------|------|------|-------|
| Idle | 60-120 | 是 | P0 |
| Walk | 30-60 | 是 | P0 |
| Run | 20-40 | 是 | P0 |
| Jump | 20-40 | 否 | P0 |
| Attack_01 | 15-30 | 否 | P0 |
| Attack_02 | 15-30 | 否 | P1 |
| Hurt | 10-20 | 否 | P0 |
| Death | 30-60 | 否 | P0 |
| Skill_01 | 20-40 | 否 | P1 |

动画混合树参数：
- Speed（移动速度）→ Idle↔Walk↔Run
- Direction（转向）→ 左转/直走/右转
- Airborne（空中状态）→ 上升/下落
```

#### Step 4: 场景搭建
```
场景搭建流程：

1. 白盒阶段（Greybox）
   - 用基础几何体搭建关卡
   - 确定比例和流线
   - 测试游戏性

2. 替换阶段
   - 逐步替换白盒为正式模型
   - 优先替换玩家接触最近的物件

3. 打光阶段
   - 设置主光源（太阳/月光）
   - 添加补光（Fill Light）
   - 设置环境光（Ambient）
   - 添加点光源（火把、灯）

4. 后处理
   - Bloom（泛光）
   - Color Grading（调色）
   - Vignette（暗角）
   - DOF（景深）
   - SSAO（屏幕空间AO）

光源设置模板：
| 光源 | 类型 | 强度 | 色温 | 用途 |
|------|------|------|------|------|
| 主光 | Directional | 1.0 | 5500K | 太阳 |
| 补光 | Directional | 0.3 | 7000K | 天空 |
| 背光 | Directional | 0.5 | 4000K | 轮廓 |
| 环境光 | Ambient | 0.2 | - | 底部填充 |
```

### 3D工具推荐
| 工具 | 用途 | 价格 |
|------|------|------|
| Blender | 建模/动画/渲染 | 免费 |
| Substance Painter | 材质绘制 | 订阅 |
| ZBrush | 高模雕刻 | 订阅/买断 |
| Mixamo | 自动骨骼绑定 | 免费 |
| Simplygon | 自动LOD | 免费 |

## 四、动画制作（Animation）

### 职责定义
制作游戏内所有动态表现，包括角色动画、UI动画、特效动画、过场动画。

### 关键产出物
- **动画控制器（Animator Controller）**：状态机与混合树
- **动画片段（Animation Clips）**：各动作的帧数据
- **动画事件（Animation Events）**：触发音效/特效的时机

### 详细操作步骤

#### Step 1: 搭建动画状态机
```
Unity Animator Controller 结构：

                ┌─────────┐
         Any    │  Entry  │
        State ──►─────────┘
          │           │
          │      ┌────▼────┐
          │      │  Idle   │◄──────┐
          │      └────┬────┘       │
          │      Speed>0.1         │ Speed<0.1
          │      ┌────▼────┐       │
          │      │  Walk   │───────┘
          │      └────┬────┘
          │      Speed>0.5
          │      ┌────▼────┐
          │      │   Run   │
          │      └─────────┘
          │
  OnDamage ┌────▼────┐
           │  Hurt   │ ──► Idle
           └─────────┘
  HP<=0  ┌────▼────┐
          │  Dead   │
          └─────────┘
```

#### Step 2: 动画事件配置
```
动画事件标记点：

攻击动画事件：
帧0:  开始（不可打断）
帧5:  "weapon_enable"  ← 开启攻击判定
帧10: "hit"            ← 伤害生效帧
帧12: "weapon_disable" ← 关闭攻击判定
帧20: "can_combo"      ← 可衔接连招
帧25: 结束（可打断）

音效事件：
帧5:  "sfx:swing_01"
帧10: "sfx:hit_01"
```

#### Step 3: UI动画制作
```
UI动画类型：

1. 过渡动画（Transition）
   - 淡入淡出：Alpha 0→1 / 1→0，持续0.3s
   - 滑入滑出：从屏幕外滑入，0.3s，EaseOut
   - 缩放弹出：Scale 0→1.1→1，0.3s，Overshoot

2. 反馈动画（Feedback）
   - 按钮按下：Scale 1→0.95→1，0.1s
   - 受伤闪红：Color闪红，0.1s
   - 金币获得：+10飘字向上，0.5s

3. 循环动画（Loop）
   - 呼吸效果：Scale 1→1.02→1，2s循环
   - 光晕旋转：Rotation 0→360，5s循环
   - 粒子拖尾：持续发射

DOTween代码示例：
// 按钮按下
transform.DOScale(0.95f, 0.1f).SetLoops(2, LoopType.Yoyo);

// 淡入
canvasGroup.DOFade(1f, 0.3f).SetEase(Ease.OutQuad);

// 飘字
text.rectTransform.DOMoveY(targetY, 0.5f);
text.DOFade(0f, 0.5f).SetDelay(0.3f);
```

## 五、资源管理与优化（Asset Management）

### 职责定义
管理所有美术资源的命名、格式、压缩、打包、版本控制。

### 关键产出物
- **资源命名规范**：统一的命名规则
- **图集（Atlas）**：合并小图为大图
- **资源预算表**：各类型资源的内存/包体预算

### 详细操作步骤

#### Step 1: 命名规范
```
命名规则：[类型]_[模块]_[名称]_[变体]

类型前缀：
  char_  角色（char_hero_idle_01）
  env_   环境（env_tree_oak_01）
  ui_    界面（ui_btn_start_default）
  fx_    特效（fx_explosion_fire_01）
  icon_  图标（icon_item_potion_hp）
  tex_   贴图（tex_ground_grass_01）
  mat_   材质（mat_metal_rusty）
  snd_   音效（snd_hit_sword_01）
  mus_   音乐（mus_boss_phase2）

变体后缀：
  _01, _02  序号
  _v2       版本号
  _wip      制作中
```

#### Step 2: 图集打包
```
TexturePacker / Unity Sprite Atlas 设置：

图集尺寸：1024×1024 或 2048×2048
间距（Padding）：2-4px（防止出血）
算法：MaxRects（最紧密排列）

分组策略：
  atlas_char_hero.png    ← 主角所有动画帧
  atlas_char_enemy.png   ← 敌人动画帧
  atlas_env_forest.png   ← 森林场景瓦片+装饰
  atlas_ui_common.png    ← 通用UI元素
  atlas_fx_common.png    ← 通用特效帧
```

#### Step 3: 资源预算
```
移动端预算参考（单场景）：

| 资源类型 | 内存预算 | 包体预算 |
|---------|---------|---------|
| 贴图总量 | ≤80MB | ≤50MB |
| 模型总量 | ≤20MB | ≤10MB |
| 音频总量 | ≤30MB | ≤20MB |
| 动画数据 | ≤10MB | ≤5MB |

PC预算参考（更宽松）：
| 资源类型 | 内存预算 |
|---------|---------|
| 贴图总量 | ≤500MB |
| 模型总量 | ≤100MB |
| 音频总量 | ≤200MB |

压缩格式：
| 平台 | 贴图压缩 | 音频压缩 |
|------|---------|---------|
| PC | BC7/DXT5 | Vorbis |
| Android | ETC2/ASTC | Vorbis |
| iOS | ASTC | AAC/MP3 |
| WebGL | DXT/ETC2 | Vorbis |
```

#### Step 4: 资源检查清单
```
导入引擎前的检查：

- [ ] 命名是否符合规范？
- [ ] 尺寸是否为2的幂（256/512/1024/2048）？
- [ ] 是否有无用的透明像素？
- [ ] 贴图是否设置了正确的压缩格式？
- [ ] 模型面数是否在预算内？
- [ ] 动画帧率是否统一？
- [ ] 材质是否使用了正确的Shader？
- [ ] 是否已打成图集？
- [ ] 文件大小是否合理？
```
