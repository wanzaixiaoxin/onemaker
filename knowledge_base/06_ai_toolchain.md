# 06 AI工具链串联总览（AI Toolchain Integration）

## 概述

作为独立游戏开发者，AI是你最强大的"虚拟团队"。本文档将梳理如何用AI工具链串联游戏开发的全流程，实现一人高效产出。

## 二、Prompt工程库

### 核心Prompt模板

#### 2.1 游戏设计类Prompt

```
【概念生成】
你是一个资深游戏设计师。请为以下需求生成游戏概念：
- 游戏类型：[类型]
- 核心机制：[一句话描述]
- 目标平台：[平台]
- 参考游戏：[参考作品]

请输出：
1. 一句话概念（≤30字）
2. 核心循环图（文字描述）
3. 3个独特卖点
4. 潜在风险和应对方案

---

【关卡设计】
为我的[游戏类型]游戏设计一个[关卡类型]关卡：
- 游戏机制：[已有机制列表]
- 难度定位：[简单/中等/困难]
- 关卡时长：[分钟]

请输出：
1. 关卡布局描述（区域划分）
2. 节奏曲线（紧张度变化）
3. 敌人/障碍分布
4. 隐藏要素位置
5. 教学点安排

---

【数值平衡】
为我的[RPG/动作]游戏设计数值系统：
- 最大等级：[X]
- 期望游戏时长：[X]小时
- 核心属性：[力量/敏捷/智力等]

请输出：
1. 属性成长公式
2. 伤害计算公式
3. 等级-经验值曲线
4. 装备数值范围表（前5级示例）
5. 经济系统参数
```

## 三、AI辅助开发工作流

### 3.1 按开发阶段的AI使用流程

```
【概念期 AI工作流】

Day 1: 创意探索
  AI对话 → 生成10个游戏概念 → 筛选Top3
  AI图像 → 为Top3生成概念图 → 选定方向

Day 2-3: 概念深化
  AI对话 → 完善核心循环、USP
  AI图像 → 生成角色/场景概念图
  AI对话 → 可行性评估（技术/时间/预算）

Day 4-5: 文档产出
  AI对话 → 生成概念文档
  AI对话 → 生成GDD初稿
  人工审核 → 修改确认

---

【原型期 AI工作流】

Week 1: 技术验证
  AI代码 → 生成核心架构代码
  AI代码 → 实现最小可玩原型
  AI对话 → 解决技术难点

Week 2: 原型迭代
  测试原型 → 发现问题
  AI代码 → 快速修复和迭代
  AI图像 → 占位美术资源
  AI对话 → 设计方案优化

---

【生产期 AI工作流】

每日循环：
  上午：AI辅助编码/美术
    → AI生成代码框架 → 人工精修
    → AI生成美术素材 → 人工筛选
  下午：内容制作
    → AI辅助数值设计
    → AI辅助关卡设计
    → AI生成音效/BGM
  晚上：测试+总结
    → AI辅助Bug分析
    → AI对话回顾当日进度

---

【发行期 AI工作流】

发行前：
  AI文案 → 商店描述文案
  AI图像 → 商店素材（封面/截图标注）
  AI音乐 → 预告片配乐
  AI对话 → 营销策略建议

发行后：
  AI对话 → 分析玩家反馈
  AI文案 → 更新公告
  AI对话 → DLC/更新规划
```

## 四、AI工具操作教程

### 4.1 Midjourney/Stable Diffusion 快速上手

```
Midjourney操作流程：
1. 加入Discord服务器
2. 在频道输入 /imagine prompt: [你的描述]
3. 等待生成（约1分钟）
4. 选择U1-U4放大 或 V1-V4变体
5. 下载图片

关键参数：
--ar 16:9    宽高比
--q 2        质量
--s 750      风格化程度
--no text    排除元素

Stable Diffusion本地部署：
1. 安装Python 3.10+
2. 克隆webui: git clone [stable-diffusion-webui]
3. 运行 webui.bat
4. 下载模型到models/Stable-diffusion/
5. 在Web界面输入Prompt生成

推荐模型：
- 二次元：Anything V5、Counterfeit
- 写实：Realistic Vision、ChilloutMix
- 像素：Pixel Art模型
```

### 4.2 Suno AI 音乐生成

```
Suno操作流程：
1. 访问suno.com，登录账号
2. 点击Create
3. 输入描述或选择风格
4. 等待生成（约1-2分钟）
5. 下载音频文件

优化技巧：
- 明确指定BPM和调性
- 使用"instrumental"排除人声
- 添加"loopable"关键词
- 用"video game soundtrack"限定风格
- 生成多个版本择优

后处理：
- Audacity裁剪到循环点
- 添加淡入淡出
- 调整EQ适应游戏
- 导出为OGG格式
```

### 4.3 Edge TTS 语音合成

```
Edge TTS使用教程：

安装：
pip install edge-tts

基础用法：
edge-tts --text "你好，冒险者！" --voice zh-CN-YunxiNeural --write-media hello.wav

中文语音选择：
| 语音名 | 性别 | 风格 |
|--------|------|------|
| zh-CN-YunxiNeural | 男 | 阳光少年 |
| zh-CN-XiaoxiaoNeural | 女 | 甜美女声 |
| zh-CN-YunjianNeural | 男 | 沉稳男声 |
| zh-CN-XiaoyiNeural | 女 | 活泼少女 |
| zh-CN-YunyangNeural | 男 | 新闻播音 |

批量生成脚本：
import edge_tts, asyncio

async def gen(text, out, voice="zh-CN-YunxiNeural"):
    comm = edge_tts.Communicate(text, voice)
    await comm.save(out)

lines = [
    ("看招！", "snd_vo_attack_01.wav"),
    ("受死吧！", "snd_vo_attack_02.wav"),
    ("啊！", "snd_vo_hurt_01.wav"),
]

for text, out in lines:
    asyncio.run(gen(text, out))
```

## 五、AI使用的局限与注意事项

### 5.1 AI的局限性

| 局限 | 表现 | 应对策略 |
|------|------|---------|
| 上下文遗忘 | 长对话后忘记之前约定 | 每次对话提供关键上下文摘要 |
| 代码幻觉 | 生成不存在的API调用 | 验证每个API是否真实存在 |
| 风格不一致 | 图像生成风格不统一 | 固定种子/使用LoRA/保持Prompt结构 |
| 数值不准 | 数值平衡建议可能不合理 | 作为起点，通过测试验证 |
| 创意趋同 | 生成内容偏向主流 | 用独特关键词引导，人工筛选 |
| 版权风险 | 不确定生成内容的版权 | 避免提及具体IP，自行验证 |

### 5.2 版权与合规

```
AI生成内容的版权注意事项：

1. 代码
   - AI生成的代码一般可自由使用
   - 但需检查是否抄袭开源项目
   - 关键算法建议理解后自己重写

2. 图像
   - Midjourney：付费用户可商用
   - Stable Diffusion：一般可商用（取决于模型许可）
   - 建议用AI生成概念图，正式素材人工制作

3. 音乐
   - Suno：付费计划可商用
   - 建议AI生成灵感，正式音乐人工/购买

4. 最佳实践
   - AI用于原型和灵感
   - 正式发布内容人工把关
   - 记录AI使用情况
   - 保留所有原始文件
```

## 六、AI工具链效率最大化策略

### 6.1 一日高效工作流

