# 07 游戏发行运营通用手册（Universal Publishing Guide）

## 概述

本文档是所有游戏通用的发行运营参考手册，按**渠道×玩法×商业模式**三个维度整理所需材料、准备工作和注意事项。无论你做什么类型的游戏，都可以在此找到对应的发行准备清单。

## 二、商业模式（Business Models）

### 2.1 商业模式对比

| 模式 | 收入来源 | 适合类型 | 优势 | 劣势 | 关键指标 |
|------|---------|---------|------|------|---------|
| **买断制** | 一次购买 | 全类型 | 简单、口碑驱动 | 收入天花板 | 销量、好评率 |
| **免费+内购(F2P)** | 道具/货币/通行证 | 手游、网游 | 用户基数大 | 需持续运营 | ARPU、留存率、LTV |
| **广告变现** | 激励视频/插屏 | 休闲、小游戏 | 门槛低 | 体验受损 | eCPM、展示率 |
| **订阅制** | 月/年费 | 服务型、内容型 | 稳定收入 | 内容压力大 | 订阅数、流失率 |
| **混合模式** | 购买+DLC+内购 | 中大型游戏 | 收入多元 | 复杂度高 | 各项综合 |
| **众筹** | 预售/赞助 | 创新型 | 提前验证 | 交付压力 | 众筹金额 |
| ** Patreon/赞助** | 月度赞助 | 持续更新型 | 直接收入 | 规模有限 | 赞助人数 |

## 三、游戏类型运营要点（Game Genre Operations）

### 3.1 各类型运营重点

| 类型 | 核心运营指标 | 关键运营动作 | 内容更新方向 |
|------|------------|-------------|-------------|
| **动作/ACT** | 通关率、连招使用率 | Boss挑战赛、速通排行 | 新Boss/武器/难度模式 |
| **RPG** | 平均游玩时长、任务完成率 | 角色投票、剧情讨论 | 新剧情/角色/装备 |
| **Roguelike** | 单局时长、重玩率 | 每日挑战、种子分享 | 新遗物/角色/楼层 |
| **模拟经营** | 日活、在线时长 | 限时活动、合作玩法 | 新建筑/场景/季节 |
| **策略/SLG** | 留存率、对战频率 | 赛季天梯、锦标赛 | 新兵种/地图/模式 |
| **平台跳跃** | 通关率、死亡率 | 速通排行、自制关卡 | 新关卡/皮肤/机制 |
| **射击/STG** | DAU、匹配时间 | 排位赛、限时模式 | 新地图/武器/角色 |
| **解谜** | 通关率、卡关点 | 每日谜题、提示系统 | 新关卡/机制/主题 |
| **视觉小说** | 完读率、分支探索率 | 角色人气投票 | 新路线/CG/番外 |
| **放置/Idle** | 留存率、在线时长 | 限时加速、签到活动 | 新系统/阶段/角色 |
| **多人竞技** | CCU、匹配速度 | 赛季排位、锦标赛 | 平衡性调整/新角色 |
| **沙盒/建造** | 创作活跃度、分享率 | 创作比赛、推荐精选 | 新方块/机制/工具 |

### 3.2 不同类型的内容更新节奏
```
单机买断制更新节奏：
  小更新：每1-2月 → Bug修复+小优化
  中更新：每2-3月 → 新模式/新关卡
  大更新：每4-6月 → 大型DLC

F2P持续运营节奏：
  每日：签到奖励、每日任务
  每周：限时活动、周常任务
  每月：月度通行证、主题活动
  每季度：大版本更新、新赛季
  每半年：周年庆活动

Live Service运营日历模板：
  1月：新年活动
  2月：春节/情人节活动
  3月：春季更新
  4月：周年庆（假设4月发行）
  5月：五一活动
  6月：年中大促+夏季更新
  7月：暑期活动
  8月：暑假末期冲刺
  9月：秋季更新
  10月：国庆活动
  11月：感恩节/黑色星期五促销
  12月：圣诞/年末活动+冬季大更新
```

### 4.3 知识产权保护
```

- [ ] 软件著作权登记（国内）

- [ ] 美术作品版权登记

常见版权陷阱：
- 字体：微软雅黑等不可商用 → 用思源/站酷免费字体
- 音乐：AI生成音乐商用需确认平台许可
- 图片：网络图片不可直接使用
- 代码：GPL协议代码可能要求开源
- 素材：免费素材需确认商用授权
```

## 五、数据分析体系（Analytics Framework）

### 5.1 通用数据指标
```
发行前指标：
| 指标 | 来源 | 目标 |
|------|------|------|
| 愿望单数 | Steamworks | >5000(发行前) |
| 预约数 | TapTap/官网 | >10000 |
| 社群人数 | QQ群/Discord | >500 |
| 社媒粉丝 | B站/Twitter | >2000 |

发行后核心指标：
| 指标 | 公式 | 健康值 | 记录频率 |
|------|------|-------|---------|
| 日销量 | 当日销售份数 | 趋势上升 | 每日 |
| 累计销量 | 求和 | - | 每日 |
| 好评率 | 好评数/总评数 | >80% | 每日 |
| 退款率 | 退款/购买 | <10% | 每周 |
| DAU | 日活跃玩家 | 稳定/增长 | 每日 |
| 在线峰值 | 同时在线最高 | - | 每日 |
| 平均游玩 | 平均单次游玩时长 | >30min | 每周 |
| 完成率 | 通关人数/购买人数 | >20% | 每月 |
```

### 5.2 按商业模式的KPI
```
买断制KPI：
- 总销量
- 总收入（扣除平台抽成）
- 好评率
- 媒体评分
- Twitch/YouTube曝光量

F2P KPI：
- 注册用户数
- DAU / MAU
- 付费率（Paying Rate）
- ARPU（每用户平均收入）
- ARPPU（每付费用户平均收入）
- LTV（用户生命周期价值）
- CAC（用户获取成本）
- LTV/CAC > 3

广告变现KPI：
- 展示量（Impressions）
- 点击率（CTR）
- eCPM（千次展示收入）
- 广告ARPU
- 填充率（Fill Rate）
```
