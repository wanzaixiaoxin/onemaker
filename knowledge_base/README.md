## 游戏制作全流程要素总览

游戏制作可以拆分为 **5大领域、18个子模块**，一人独立开发时每个模块都需覆盖：

### 一、游戏设计（Game Design）
| 子模块 | 职责 | 关键产出物 |
|--------|------|-----------|
| 概念设计 | 确定游戏核心概念、类型、目标受众 | 概念文档 |
| 核心玩法 | 设计核心循环、操作方式、难度曲线 | GDD(游戏设计文档) |
| 关卡设计 | 设计关卡布局、节奏、流程 | 关卡白盒/流程图 |
| 系统设计 | 设计经济系统、成长系统、社交系统等 | 系统规格文档 |
| 叙事设计 | 剧情、对话、世界观、角色设定 | 剧本文档 |

### 二、程序开发（Programming）
| 子模块 | 职责 | 关键产出物 |
|--------|------|-----------|
| 引擎/框架选型 | 选择合适的游戏引擎和工具链 | 技术选型文档 |
| 核心架构 | 搭建游戏框架、模块划分、数据流 | 代码架构 |
| 玩法实现 | 实现核心玩法逻辑、物理、碰撞 | 游戏逻辑代码 |
| UI/UX开发 | 菜单、HUD、交互反馈 | UI系统 |
| 工具链/管线 | 构建自动化、资源管线、编辑器扩展 | 工具脚本 |

### 三、美术资源（Art）
| 子模块 | 职责 | 关键产出物 |
|--------|------|-----------|
| 视觉风格定义 | 确定美术方向、参考图、色彩方案 | 美术风格指南 |
| 2D/3D资产制作 | 角色、场景、道具、特效 | 图片/模型/动画 |
| UI美术 | 界面设计、图标、字体 | UI资产包 |
| 技术美术(TA) | Shader、渲染管线、性能优化 | Shader/渲染工具 |

### 四、音频（Audio）
| 子模块 | 职责 | 关键产出物 |
|--------|------|-----------|
| 音乐制作 | BGM作曲、编曲、混音 | 音乐文件 |
| 音效设计 | UI音效、环境音、战斗音效 | 音效文件 |
| 语音 | 角色配音、旁白（可选） | 语音文件 |

### 五、项目管理与发行运营（PM & Publishing）
| 子模块 | 职责 | 关键产出物 |
|--------|------|-----------|
| 项目管理 | 进度规划、任务分解、里程碑管理 | 项目计划 |
| QA测试 | 功能测试、性能测试、兼容性测试 | 测试报告 |
| 发行上架 | 平台对接、商店页面、合规审查 | 上架包 |
| 营销运营 | 宣传素材、社区运营、数据分析 | 营销方案 |

## 独立开发者AI协作原则

1. **AI是团队**：一个人 + AI = 一个团队。把AI视为各领域的专家顾问
2. **文档先行**：每个模块先让AI帮你写规格文档，再动手实现
3. **迭代驱动**：小步快跑，每次只聚焦一个子模块
4. **工具链串联**：用AI打通各模块间的数据流和工作流
